# Documentation Layer

This directory serves as the **Single Source of Truth** for all raw scientific content, research documents, and assets provided by the domain experts.

## Folder Responsibilities

### `/platform`
Contains platform-level information and meta-documentation.
- **What belongs here**: Platform Overview, About Platform, Research Information, Researcher Name, Supervisors, Objectives, Target Audience.
- **Naming Convention**: Use descriptive PascalCase or Title Case names.
- **Supported Formats**: `.docx`, `.pdf`
- **Example**: `About-Platform.docx`, `Target-Audience.pdf`

### `/modules`
Contains the core scientific modules and course material.
- **What belongs here**: The original scientific text documents detailing the learning content.
- **Naming Convention**: `Module-XX.docx` or descriptive names.
- **Supported Formats**: `.docx`
- **Example**: `Module-01.docx`, `Module-02.docx`

### `/assessments`
Contains all testing and evaluation material.
- **What belongs here**: Pre Assessment, Post Assessment, Answer Keys, Evaluation Rubrics.
- **Naming Convention**: Descriptive names separated by hyphens.
- **Supported Formats**: `.docx`, `.xlsx`, `.pdf`
- **Example**: `Pre-Assessment.docx`, `Post-Assessment.docx`, `Answer-Key.docx`

### `/references`
Contains external literature and citations.
- **What belongs here**: Scientific References, Books, Research Papers, External Resources.
- **Naming Convention**: `Author-Year-Title.pdf` or descriptive names.
- **Supported Formats**: `.pdf`, `.docx`, `.txt`
- **Example**: `Smith-2023-Cognitive-Load.pdf`

### `/assets`
Contains all visual and static media utilized in the scientific documents.
- **What belongs here**: Logos, Images, Icons, Scientific Figures.
- **Naming Convention**: Lowercase, hyphen-separated.
- **Supported Formats**: `.png`, `.jpg`, `.svg`
- **Example**: `scientific-figure-1.png`, `platform-logo.svg`
